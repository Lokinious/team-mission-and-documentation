# Working Agreements

## Collaboration principles
- Keep communication direct, respectful, and practical
- Surface risks and blockers early
- Prefer small, reviewable changes over large unclear ones
- Document decisions that affect architecture, process, cost, or scope

## Documentation expectations
- Important project decisions should be written down
- New repositories should include a clear README
- Plans and work items should stay current enough to be useful

## Delivery expectations
- Focus on realistic milestones
- Protect quality without overengineering
- Call out tradeoffs between speed, polish, and maintenance cost

## Operating mindset
- We are building for sustainable progress, not just activity
- We keep learning loops short
- We keep the startup adaptable while creating enough structure to avoid chaos
