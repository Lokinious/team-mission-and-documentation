# Supporting Workstreams

## Overview
These workstreams support the entire repository portfolio. They are important because they reduce avoidable rework, improve team clarity, and create better operating discipline from the beginning.

## Highest-priority support items
1. GitHub ownership model and access defaults
2. Repository standards and starter templates
3. Security and secret-management baseline
4. Finance and bookkeeping baseline
5. Legal and policy starter checklist
6. Branding foundation
7. Release and artifact management strategy

## Expanded support workstreams

### Repository standards
- Standardize README structure
- Define naming conventions
- Add contribution and documentation expectations
- Create a reusable checklist for new repositories

### GitHub ownership and access
- Decide personal account versus organization
- Define default repository visibility and access rules
- Add branch protection and review expectations where appropriate

### Branding and content foundation
- Create a placeholder brand system
- Align visual direction across UI, launcher, and content
- Define how reviews, videos, and project updates should stay consistent

### Publishing workflow
- Define how reviews, videos, updates, and release notes are produced
- Create a lightweight editorial workflow
- Standardize metadata for content publishing

### Backend foundation
- Define the shared backend strategy for launcher manifests, multiplayer, content feeds, and future APIs
- Decide when shared services should be split into separate repositories

### Security and secrets
- Define credential handling rules
- Define secret storage expectations
- Add dependency and vulnerability scanning as repositories are created

### Release and artifact management
- Standardize versioning and release-note practices
- Decide where installer and build artifacts will live
- Define how launcher-managed downloads will be published

### Legal and policy baseline
- Prepare privacy, terms, donation, copyright, and contact checklists
- Track third-party asset and license obligations
- Keep a list of items that should later be reviewed with counsel

### Finance and operations
- Set bookkeeping-friendly categories
- Track recurring costs and subscriptions
- Define receipt and invoice storage habits
- Establish a simple monthly financial review routine

### Analytics, observability, and quality
- Define minimum monitoring expectations
- Define basic analytics goals
- Standardize linting, testing, formatting, and CI expectations per repository type

### Roadmap and decision tracking
- Keep major decisions visible
- Record deferred items instead of losing them
- Review roadmap priorities as the startup evolves
